# 🚀 Day 1 – Linux Basics & Git Fundamentals

> **Goal:** Build a strong foundation in Linux and Git, which are the core skills required for every DevOps Engineer.

---

# 📖 Overview

Day 1 focused on understanding the Linux operating system, learning essential Linux commands, and setting up Git for version control. These are the first building blocks for a successful DevOps career.

By the end of Day 1, you should be comfortable navigating the Linux terminal, managing files and directories, monitoring basic system information, and tracking changes in a Git repository.

---

# 🎯 Learning Objectives

After completing Day 1, you should be able to:

* Understand what Linux is and why it is widely used.
* Navigate the Linux file system.
* Use essential Linux commands confidently.
* Monitor system resources.
* Create and manage files and directories.
* Understand the basics of Git and version control.
* Initialize a Git repository.
* Stage and commit changes.
* Push a local repository to GitHub.

## Essential Linux Commands

Commands practiced during Day 1 include:

```bash
pwd
ls
ls -la
cd
mkdir
touch
cp
mv
rm
cat
clear
history
```

System monitoring commands:

```bash
uptime
free -h
df -h
du -sh
```

## Git Fundamentals

Topics covered:

* What is Git?
* Why Version Control Matters
* Installing Git
* Configuring Git
* Creating a Repository
* Staging Files
* Creating Commits
* Viewing Commit History
* Connecting to GitHub
* Pushing Code

Commands used:

```bash
git init
git config --list
git status
git add .
git commit -m "message"
git branch -m main
git remote add origin <repository-url>
git push -u origin main
git log
```

# 🌍 Real-World Relevance

Linux and Git are essential tools in modern DevOps environments.

Every DevOps Engineer uses Linux to manage servers, automate tasks, and deploy applications. Git is used to track changes, collaborate with teams, and maintain project history.

Mastering these fundamentals is critical before moving on to Docker, Kubernetes, CI/CD, and cloud platforms.
