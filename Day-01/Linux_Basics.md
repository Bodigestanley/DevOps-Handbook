# 📖 Introduction

Linux is a free, open-source operating system based on Unix principles. It is widely used to power servers, cloud platforms, supercomputers, embedded systems, networking devices, and Android smartphones.

Today, Linux is the preferred operating system for DevOps engineers because of its stability, security, performance, and automation capabilities.

# 📜 Brief History of Linux

* **1969** – Unix was created at Bell Labs.
* **1991** – Linus Torvalds developed the Linux kernel.
* The Linux kernel was released as open source, allowing developers worldwide to contribute.
* Today, Linux powers millions of servers and devices across the globe.

# 🤔 What is an Operating System?

An Operating System (OS) is software that acts as a bridge between the user and the computer hardware.

Its responsibilities include:

* Managing CPU usage
* Managing memory (RAM)
* Managing files and storage
* Running programs
* Managing users and permissions
* Handling input and output devices

Examples of operating systems:

* Linux
* Windows
* macOS

---

# 🏗️ Linux Architecture

Linux can be understood as four major layers:

```text
+---------------------------+
|        User               |
+---------------------------+
|        Shell              |
+---------------------------+
|        Kernel             |
+---------------------------+
|       Hardware            |
+---------------------------+
```

### 1. User

The person interacting with the computer.

### 2. Shell

The command interpreter that receives user commands and communicates with the kernel.

Examples:

* Bash
* Zsh
* Fish

### 3. Kernel

The core of the operating system.

It is responsible for:

* Process management
* Memory management
* Device management
* File system management
* Networking

### 4. Hardware

The physical components:

* CPU
* RAM
* Hard Disk / SSD
* Keyboard
* Network Card

---

# 🧠 What is the Kernel?

The kernel is the heart of Linux.

Without it, the operating system cannot function.

Its main responsibilities include:

* Scheduling processes
* Allocating memory
* Managing hardware devices
* Handling system calls
* Managing security and permissions

---

# 💻 What is the Shell?

The shell is the interface between the user and the kernel.

When you type a command like:

```bash
ls
```

the shell:

1. Reads the command.
2. Interprets it.
3. Sends it to the kernel.
4. Displays the output.

The default shell in Ubuntu is **Bash (Bourne Again Shell)**.
